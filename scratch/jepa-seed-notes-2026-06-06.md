# JEPA Seed Notes — 2026-06-06

Messy notes, keep until cleaned.

- root problem is not "better memory", it is "less raw intent archaeology"
- chat should be ingress, not storage
- seed is probably the product word
- jepa/capture/portal are machine words
- user should be able to dump links/reels/articles/random thought chains all day
- system should auto-route to:
  - global seed area
  - candidate project buckets
  - repo if obvious
- continuation command should feel like:
  - "hey codex/claude, continue from this clarified seed"
- AI should load:
  - technical graph stream
  - current repo state
  - package anchors
  - contradictions
  - changed files since last continuation
- human should get:
  - how this fits prior work
  - what changed
  - what the next move is
- likely product naming:
  - `seed ingest`
  - `seed continue`
  - `seed refine`
- likely anti-pattern:
  - too many user-facing machine nouns
- likely moat:
  - one canonical package, many projections
- biggest benchmark:
  - does this kill the 250k-token intent recovery tax?
