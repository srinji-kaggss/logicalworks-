## What lgwks IS now (the reframe — read this first)
lgwks is the **research co-processor for the coding AI**, not a CLI toy. You (a coding agent) hand it
your **implementation guide**; while you code in the foreground, it runs an autonomous deep-research
loop on that guide in the background, grounding the plan's claims against ctx7+web, and generates
artifacts you pull as you work. It **replaces the ad-hoc grounding reflex** (one-off firecrawl/ctx7)
with a standing research substrate.

- **Autonomous loop** (`lgwks_research.py`, `lgwks-akinator --auto`): generate→crawl→reason→contrarian→
  expand→save per round. Budget-capped convergence. crawl_mode = `estimate` (planning) | `ground`
  (ctx7+web evidence) | `live` (gated spine, NOT wired). `--guide <file>` ingests a plan.

- **Spine** (`lgwks_run.py`): gated crawl (C1/C2/C3, signed verdicts, scope-frozen, IP denylist).
- **Vault** (`tools/lgwks-auth`): WORM capability log, now keyed-HMAC via `lgwks_sign.py` (not source const).
