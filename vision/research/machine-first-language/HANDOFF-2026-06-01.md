## RESIDUAL RISK (read before shipping further)

1. **The data boundary (T0) has NOT had an adversarial hacker review.** Tests cover the happy + key
   invariants (fail-closed, encrypted-at-rest, tamper-evidence, separation, no-exec) but a dedicated
   abuse pass is the recommended next gate before this is trusted with real PII.
2. **Vault key = derived from the signing secret.** Rotating the signing key makes all vault entries
   unreadable (foreign-blobв†’absent). Acceptable for sessions (re-auth) but document for users.
3. **Cache index logs source URLs** вЂ” fine for public research pages; if an auth/PII URL ever flows
   through `ground`, strip query strings first.
4. **Floor providers are all unkeyed public indexes** вЂ” can throttle together under load; `rendered`
   needs local Chromium. Capacity ceiling, not a correctness gap; the keyed seam exists, unfunded.
5. **The Machine is cold-start heuristic**, not neural вЂ” honest scaffolding that generates the corpus;
   the intelligence arrives with #17.
