# Bot Fabric CLI Wiring

Status: spec

## Purpose

Expose the full bot fabric pipeline as a single `lgwks review` verb,
and wire autonomous bot triggering to git hooks and file-watch events.

## CLI verb: `lgwks review`

```
lgwks review [--repo PATH] [--bots LIST] [--changed FILES] [--synth] [--l-budget FLOAT]
```

### What it runs

```
1. load or refresh repo graph
2. run selected bots (default: all enabled bots in plan)
3. reduce findings (U3)
4. build JEPA package (U4)
5. check artifact strength (U11)
6. optionally run synthesizer (U9) if --synth passed and U11 passes
7. emit human report (U10) to findings/report.md
8. emit machine packet to findings/machine-packet.json
9. print L score to stdout
```

### Flags

| Flag | Default | Meaning |
|---|---|---|
| `--repo` | cwd | repo root to scan |
| `--bots` | all | comma-separated list: code_hacker,slop_math,optimizer,stress |
| `--changed` | none (full scan) | comma-separated relative file paths for subset mode |
| `--synth` | off | run synthesizer after packaging |
| `--l-budget` | 0.15 | max allowed L coefficient when synthesis runs |
| `--out` | findings/ | output directory |

### Exit codes

```
0  clean run, report emitted
1  findings above high severity detected (non-zero signal)
2  artifact strength gate failed
3  internal error (bot crash, schema failure)
```

Exit code 1 is not a failure — it means the bots found something. CI should treat it as a signal, not a build break, unless policy says otherwise.

## Autonomous trigger wiring

### Git post-commit hook

```bash
# .git/hooks/post-commit
#!/bin/sh
CHANGED=$(git diff-tree --no-commit-id -r --name-only HEAD | grep '\.py$' | tr '\n' ',')
if [ -n "$CHANGED" ]; then
  lgwks review --changed "$CHANGED" --out findings/ &
fi
```

Non-blocking (backgrounded). The hook never blocks the commit.

### File-watch mode

```
lgwks review --watch [--repo PATH]
```

Watches for `.py` file changes using OS-native events (FSEvents on macOS).
On change: runs the bot fabric on the changed subset, updates findings/.

### Seed ingest trigger

When `lgwks seed ingest` runs, it automatically calls `lgwks review --changed <ingested_files>`
on any repo-local files included in the ingest.

## Bot plan integration

`lgwks review` uses a bot plan (U2 schema) if one exists at `.lgwks/bot-plan.json`.
If no plan file exists, it uses the default plan (all bots enabled, no synth, L budget 0.15).

Default plan:
```json
{
  "schema": "lgwks.bot.plan.v1",
  "plan_id": "default",
  "run_kind": "review",
  "target_repo": ".",
  "bots": [
    {"name": "code_hacker", "enabled": true},
    {"name": "slop_math", "enabled": true},
    {"name": "optimizer", "enabled": true}
  ],
  "jepa": {"enabled": true},
  "synth": {"enabled": false, "optional": true},
  "policy": {
    "allow_external_research": false,
    "branch_state_mode": "per_branch",
    "max_artifact_bytes": 10485760
  },
  "outputs": {"root": "findings/"}
}
```

## Design constraints

1. review never blocks — backgrounded by default in hook context
2. subset mode (--changed) must produce valid output without full-repo state
3. findings/ directory is branch-local (per_branch mode)
4. L score always printed to stdout regardless of --synth flag
5. all outputs are deterministic given the same inputs
