# World DB

## Purpose

Create a machine-world database that captures the causal structure of computers, software, protocols, and internet systems without relying on human prose as the source of truth.

## Why it exists

The final AI synthesizer should begin from machine reality, not from a blank prompt or a giant transcript.

## Core object types

- repo
- file
- symbol
- service
- process
- protocol
- API surface
- auth boundary
- datastore
- network path
- build step
- test
- package
- finding
- outcome
- mitigation

## Core edge types

- imports
- calls
- reads
- writes
- depends_on
- authenticates_to
- executes
- stresses
- proves
- contradicts
- mitigates
- evolves_to

## Source classes

- repo graph extraction
- deterministic bot outputs
- substrate crawl products
- package history
- selected external standards/docs

## Human-language policy

Human language is allowed as:

- a projection
- an annotation
- a query surface

Human language is not the canonical storage shape.

## Relationship to JEPA

The world DB is the substrate.
JEPA binds views into stable latent objects over that substrate.

## Relationship to the final synthesizer

The synthesizer should consume package views derived from the world DB, not freeform internet slop.
