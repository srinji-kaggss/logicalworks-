## Key Integration Patterns

### Pattern 1: Serverless Event-Driven
```
Event Source (S3, API Gateway, EventBridge)
    → Lambda (compute)
    → DynamoDB (persistence)
    → SNS/SQS (async fan-out)
    → CloudWatch (observability)
```

### Pattern 2: Container Microservices
```
ALB (ingress)
    → ECS/EKS (container orchestration)
    → VPC (isolation)
    → RDS/DynamoDB (state)
    → CloudWatch + X-Ray (observability)
```

### Pattern 3: Data Lake → Analytics
```
S3 (raw data)
    → Glue (ETL + catalog)
    → Lake Formation (governance)
    → Athena (ad-hoc query)
    → Redshift (BI warehouse)
    → QuickSight (dashboards)
```

### Pattern 4: ML Pipeline
```
S3 (training data)
    → SageMaker (training + deployment)
    → Bedrock (LLM augmentation)
    → API Gateway (model serving)
    → CloudWatch (model monitoring)
```
