## What's working

1. `lgwks workflow research <url>` — full substrate crawl → dual vectors → concepts → spawn.json
2. `lgwks workflow deep-research <url>` — same, with configurable max_pages/max_depth
3. `lgwks workflow quick-scan <url>` — max_pages=1, max_depth=0
4. `lgwks do research <url>` — routes to substrate (non-URL falls to akinator stub)
5. `lgwks crawl <url>` — delegates to `substrate.build_run(max_pages=1, max_depth=0)`
6. Concept extraction: `EC2 (Elastic Compute Cloud)` extracted as alias → `what_is("EC2")` returns definition + related
7. Multimodal module: `embed_multimodal(text, image_b64)` returns `{det, sem}`
8. Browser screenshots: `render(..., with_screenshot=True)` returns `{screenshot_b64, screenshot_mime}`

## What's NOT working (the real gaps)

| Gap | Why | What needs to happen |
|---|---|---|
| **Screenshot not consumed by substrate** | `render()` passes `with_html=True` but never `with_screenshot=True` in `_crawl_site()` | Add `with_screenshot=True` to both `render()` calls in `lgwks_substrate_crawl.py` lines 136 and 205 |
| **Image chunks not in chunks.jsonl** | `build_run()` only processes text chunks; no image chunk rows emitted | Add image chunk creation after screenshot capture, write to chunks.jsonl with `"chunk_kind": "image"` |
| **Multimodal embed NOT called in pipeline** | `build_run()` calls `embed_dual()` (text-only) for every chunk; never calls `embed_multimodal()` for images | Detect image chunks → call `embed_multimodal()` instead of `embed_dual()`; store both det + sem in vectors.jsonl |
| **9 workflows are still stubs** | `_do_code_wrapper`, `_do_govern_wrapper`, etc. → `lgwks_do` stubs; `_do_audit_trail` → `solve_provenance`; `_do_health_check` → manifest check | Each needs substrate-powered run with real artifact tree, or needs honest deprecation |
| **No human concept query CLI** | `what_is("EC2")` is in `lgwks_concept.py` but not exposed as a CLI command | Add `lgwks concept <query>` subcommand that loads concepts.json from latest run and prints human-readable output |
| **Workflow harness is not activation-steering hub** | The workflow names (research, code, govern, ship, prove, extract, compare, audit-trail, health-check, onboard, migration-check) should trigger each other based on concept activation | Design: when `prove` run finds vulnerability concepts, activate `audit-trail`; when `govern` finds policy gaps, activate `migration-check` |

## Known issues (honest fuckups)

1. **I lost focus on the workflow harness.** Spent 2 hours fixing individual bugs (spawn.json, AUP gate type mismatch, concept graph slug collision) instead of stepping back to wire the whole picture. The user had to ask "what happened to the workflow stuff" to re-center me.
2. **Browser screenshot is wired but orphaned.** `render(with_screenshot=True)` works, but `_crawl_site()` never sets it, and even if it did, the image data never becomes a chunk row. It’s a pipe with no faucet.
3. **Embedding provider cascade has a bug.** `embed_dual()` line 636: if Apple local is also unavailable, it **raises** `EmbeddingProviderUnavailable` instead of returning `{"sem": None}`. This would crash `build_run()` if all semantic providers fail. The design intent was "never block," but the code blocks.
4. **Concept graph `_relations_by_source` type mismatch.** Fixed in `export_json()` (commit `683a757`) but the data structure (`dict[str, list[ConceptRelation]]`) is confusing. `_rels_by_source` stores lists, but `_rels_by_source.values()` was iterated as if it stored single items. The fix works but the model is fragile.
5. **`lgwks_embed.py` is stale.** It builds deterministic text-only vaults. Never calls `embed_dual()`, never touches images. It should either call `embed_dual()` or be deprecated in favor of `substrate build`.
6. **`lgwks_openrouter_embed.py` only does text.** The `embed_one()` function sends `"input": text` to OpenRouter. For multimodal it must use `lgwks_multimodal.embed_multimodal()` which has the proper `image_url` payload. These two modules duplicate intent but have different purposes.
